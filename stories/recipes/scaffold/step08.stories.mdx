## Linting

Linting will allow us to make sure the format of the CSS and Javascript will follow a standard.
Linting has huge benefits and does not take long to run and setup.

**devDependencies**
```sh
npm install eslint stylelint stylelint-config-standard --save-dev
```

Once these packages are installed we need to create the configuration files



Create `.stylelintrc`
```json
{
  "extends": "stylelint-config-standard",
  "rules": {
    "color-hex-case": "lower",
    "indentation": null,
    "no-empty-source": null,
    "selector-type-no-unknown": [
      true,
      {
        "ignore": [
          "custom-elements"
        ]
      }
    ]
  }
}
```

Create `.eslintrc`
```json
{
  "env": {
    "browser": true,
    "es2022": true,
    "node": true,
    "jasmine": true
  },
  "extends": "eslint:recommended",
  "parserOptions": {
    "ecmaVersion": "latest",
    "sourceType": "module"
  },
  "rules": {
  }
}

```

finally to test this we will add new npm scripts to execute the linter.

**package.json** update

```json
"scripts": {
  "build": "./node_modules/.bin/webpack --mode production --config webpack.config.js",
  "build:dev": "./node_modules/.bin/webpack --mode development --config webpack.config.js",
  "lint:css": "./node_modules/.bin/stylelint './src/**/*.css' --config ./.stylelintrc",
  "lint:js": "./node_modules/.bin/eslint --fix -c ./.eslintrc --ext .js ./src",
  "lint": "npm run lint:css && npm run lint:js",
  "pretest": "npm run lint",
  "start": "./node_modules/.bin/webpack serve --mode development --config webpack.config.js",
  "test": "echo \"Error: no test specified\" && exit 1"
},
```

> Note that we added the `--fix` configuration for eslint.
